## Code Standards

### Naming
- No acronyms or abbreviations in variable, function, or component names
- No single-letter variables (use descriptive names that convey purpose)
- Names should read like plain English and be self-documenting

### Comments
- Short, helpful comments that explain *why*, not *what*
- Every function and component should have a brief comment describing its purpose

### Structure
- Prefer smaller, focused components over large monolithic ones
- Each component/function should do one thing well
- Prioritize clarity over cleverness
